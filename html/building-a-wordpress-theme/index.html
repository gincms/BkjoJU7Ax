Building a Wordpress theme

<p>One of the first things people do when they decide to setup a WordPress site is go hunting for a theme. Even though I have no intention of using a pre-built theme for this blog — besides the temporary use of the Twenty Fifteen theme while I build one — I still found myself looking for a <em>starter</em> theme. The hope was that I could give myself a head start.</p> <p>Unfortunately, there are a few problems with these starter themes:</p> <ul> <li>Even the <a href="https://themes.svn.wordpress.org/blankslate/4.0.2/functions.php">barest starter</a> themes contain more code than I’d want in a starter theme</li> <li>They have a file structure I don’t appreciate (i.e. little-to-no <a href="https://developer.wordpress.org/themes/basics/linking-theme-files-directories/">file organisation</a>)</li> <li>They don’t <a href="https://www.google.co.uk/webhp?sourceid=chrome-instant&amp;ion=1&amp;espv=2&amp;ie=UTF-8#q=clean%20up%20wordpress%20header">trim</a> any of the fat that often hangs loosely from the head section of WordPress themes</li> </ul> <p>The point: the starter themes start in the <em>wrong</em> place (for my purposes). Besides, the theme I’m going to build for this site will likely be far from conventional. For these reasons, I’m going to skip the starter all together and just build from scratch, as originally intended.</p>
